# GitHub Discoverability Playbook

This file lists high-impact actions that improve repository visibility and ranking signals on GitHub.

## 1) Repository Settings (manual)

Set a keyword-rich description in GitHub repo settings:
- Example: `SQL for Data Engineering roadmap with notebooks, practice problems, CTEs, joins, window functions, and production SQL patterns.`

Set topics (recommended):
- `sql`
- `data-engineering`
- `analytics-engineering`
- `postgresql`
- `notebooks`
- `portfolio`
- `learning-roadmap`

## 2) README Optimization

Already implemented in this repo:
- Clear keyword targeting for SQL/data engineering
- Learning path and notebook order
- Progress checklists and roadmap links
- CI badge for trust signals

## 3) Activity Signals

- Keep a weekly commit cadence.
- Use descriptive PR titles tied to roadmap milestones.
- Keep roadmap checklist current so visitors see momentum.

## 4) Content Signals

- Add concise writeups to notebook introductions.
- Include practical problem framing and solution summaries.
- Prefer reproducible examples with small sample data.

## 5) Community Signals

- Keep issue templates active for structured feedback.
- Encourage contributions through `CONTRIBUTING.md`.
- Use Discussions (if enabled) for learning logs and Q&A.
